# transcription への Silero VAD スキップゲート

## 問題

RVC(vc)は非音声入力からもフルスケールの人声風ノイズを生成するため、ADR-0019 で Silero VAD による絶対判定ゲートを導入し、音声比率の低いチャンクを推論前にスキップしている。しかし同じ録音チャンクは `routes_list` で transcription にも並列に流れるのに、transcription 側には同等の門番がない。recording の門番は発話単位の dBFS 閾値(`silence_threshold`)なので、環境音・ブレス・発話後の余韻でトリガーされたチャンクがそのまま音声認識に届く。無音・ノイズ入力に対して whisper は実在しないテキスト(カタカナ等)を幻聴し、かつ無駄な GPU 推論が走る。判定を録音側で強化すると、transcription が残したいフィラー・感嘆詞まで落ちてしまうため、recording 側ではなく transcription パス側で対策する必要がある。

## ゴール

- transcription が、音声をほとんど含まないチャンクに対して音声認識をスキップできる(opt-in)。
- 判定はマイクゲインに依存しない絶対判定で、recording の dBFS 閾値とは独立に働く。
- ACP / GCP / WHISPER いずれのバックエンドでも同様に効く。
- 既定では無効で、導入前と挙動が変わらない。

## 非ゴール

- vc 側の VAD 挙動(スキップ判定・出力ダック)および設定の変更。現状維持。
- transcription 出力側の音声加工(ダック/ゲイン)。transcription は音声を出さないので、ゲートは「通す/落とす」の二値のみ。
- 新しいルーティング可能ワーカーや wire 契約(protobuf)の追加・変更。
- vc と transcription で VAD 感度設定を共有する仕組み。設定は各ワーカーで独立とする。
- whisper の既存 `no_speech` / `logprob` 閾値、および recording の `silence_threshold` の変更。

## 受入基準

- transcription に opt-in の VAD スキップゲートがあり、既定では無効である。
- 有効時、音声比率が閾値未満のチャンクは音声認識に回されず、下流(字幕・翻訳)にも出力されない。
- 有効時、十分な音声を含むチャンクは従来どおり認識され出力される。
- スキップ判定はマイク音量(ゲイン)の大小に依存しない。
- ゲートは ACP / GCP / WHISPER のどのバックエンドでも同様に効く。
- VAD モデルが見つからない、またはロードできない場合、起動時に明確に失敗する。
- スキップ判定の推論中に例外が起きた場合、そのチャンクはゲートせず音声認識に通す(発話を落とす方向には壊れない)。
- スキップされたことが観測可能(テレメトリ等)に記録される。
- transcription 側の VAD 設定は vc 側とは独立に指定できる。
- vc 側の VAD 挙動および設定は本変更で変わらない。
- ゲート無効時、本変更の前後で transcription の挙動は同一である。
