# RVC 入力音声エンベロープ転写の再設計 設計書

## 問題

RVC ボイスチェンジ後の出力に入力音声の音量変化（エンベロープ）を反映させる機能が、「入力の抑揚を忠実に写す」という目的に対して逆の挙動になっている。小さい音量を持ち上げてダイナミクスを平坦化し、出力レベルがマイクゲイン・声量の絶対値に丸ごと連動し、ゲインが階段状に飛んで振幅変調ノイズが乗り、入力無音でも出力が漏れる。さらに、その後の実機確認で、入力の相対抑揚をそのまま乗じると声の大きい瞬間に出力がクリップすることも判明した。

## ゴール

- 入力の *相対的な抑揚の形* を、RVC 出力本来の平均レベルを保ちながら滑らかに転写する。
- マイクゲイン（入力の絶対レベル）に左右されない、頑健な音量整合にする。
- 出力がどの瞬間もクリップしない。

## 非ゴール

- RVC 推論本体（ボイス変換）の経路変更。
- 入力ブースト処理の変更。
- マルチチャンネル対応（出力は mono 固定）。
- 設定の後方互換 alias（クリーン入れ替えでよい）。

## 受入基準

- [ ] 一定振幅の入力では、出力の平均レベルが RVC 出力とほぼ一致する（レベル保存）。
- [ ] 入力音量が時間方向に増減すると、出力の音量変化がその抑揚の形をなぞる。
- [ ] マイクゲイン（入力の絶対レベル）を変えても、抑揚の形が同じなら出力の抑揚は変わらない。
- [ ] 出力の音量変化は滑らかで、振幅変調由来のノイズ（zipper noise）が乗らない。
- [ ] 出力がどの瞬間もクリップしない（ゲインはダック方向＝出力を下げる向きのみ）。
- [ ] 入力が実質無音のチャンクでは、出力が RVC 出力のまま変化しない（ノイズを持ち上げない）。
- [ ] 転写の深さを 0 にすると、出力が RVC 出力と一致する（無効化）。
- [ ] 出力の長さは常に RVC 出力と一致する。
