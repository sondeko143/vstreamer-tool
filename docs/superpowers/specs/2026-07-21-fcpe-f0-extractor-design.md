# FCPE を低レイテンシ f0 抽出器として追加

## 問題

RVC(vc) の音声変換は f0(基本周波数)抽出を必要とし、既定の rmvpe は精度に優れる一方で大きめのモデルを GPU 上で ONNX 実行する。f0 抽出は実時間 VC 経路のレイテンシに乗る処理であり、ここを短縮できればエンドツーエンド遅延に余地が生まれる。FCPE はより小型で高速なピッチ推定器で、rmvpe と同じ「16k 波形入力 → f0」の契約に載せられる見込みがあるが、現状このパイプラインには FCPE を使う手段が無い(パイプラインの波形入力契約に合う ONNX 資産が存在せず、抽出器として選択もできない)。

## ゴール

- FCPE を f0 抽出器として選択でき、rmvpe より低い f0 抽出レイテンシで RVC(vc) を動かせる。
- FCPE が既存の rmvpe/dio/harvest と同じ f0 抽出経路に載り、設定の切り替えだけで使える。
- FCPE の ONNX 出力が torch 参照実装と数値的に一致することを検証できる。

## 非ゴール

- rmvpe を既定から置き換えること。FCPE はオプトインとし、既定は rmvpe のまま。
- FCPE の全パラメータ(デコード方式・f0 下限/上限・threshold)を実行時に可変化すること。生成時に固定してよい。
- 事前生成した FCPE の ONNX 資産を配布物へ同梱/再配布すること。今回はオフライン生成のみ。
- dio / harvest / rmvpe の既存挙動、および wire 契約(protobuf)・ルーティングの変更。
- transcription 側 VAD や vc 以外のワーカーへの波及。

## 受入基準

- FCPE を選んだときの f0 抽出レイテンシが、同一音声・同一条件の rmvpe より低いことが実測ベンチで記録されている(これを満たせない場合 FCPE は出荷しない)。
- 設定で f0 抽出器として FCPE を選べる。
- FCPE を選んで RVC(vc) がクラッシュせず音声変換を行い、出力が f0 として妥当(無音でも破綻でもない)である。
- FCPE の ONNX 資産が存在しない/不正なとき、起動前チェックが「どの設定を直すべきか」を示して失敗する。
- FCPE の ONNX 出力が torch 参照実装と許容誤差内で一致することがテストで確認できる。
- FCPE の ONNX 資産をオフラインで生成する手順がドキュメント化されている。
- FCPE 追加の前後で、dio / harvest / rmvpe を選んだときの挙動は同一である。
- 既定(rmvpe)のまま使う限り、本変更の前後で挙動は変わらない。
