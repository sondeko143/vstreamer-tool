# VC VAD ノイズゲート設計 (2026-07-08)

## 問題

RVC は非音声入力（環境音・ブレス）からもフルスケールの人声風ノイズを生成する。漏れ道は 2 つ。(1) ノイズ単独チャンク — 録音の門番が発話単位の音量閾値なので、環境音・ブレスだけで録音がトリガーされるとチャンク全体がノイズのまま VC に届き、相対エンベロープは平均正規化のため gain≈1 に正規化して素通しする。(2) 発話内のブレス・環境音 — 発話中は全区間が素通しで、録音前のプリロールも先頭に付く。両方の症状がユーザーで確認済み。

## ゴール

- 非音声チャンク全体からのフルスケールノイズを塞ぐ。
- 発話内のブレス・環境音区間を出力側で抑える。
- マイクゲインの絶対値に左右されない絶対判定にする。

## 非ゴール

- 録音側の閾値変更（並列に流れる transcription のフィラー・感嘆詞を落とすため）。
- transcription パスへの影響。
- ノイズ抑制前処理（発話に重なる定常ノイズ対策）— 症状ではないため今回はしない。

## 受入基準

- [ ] ゲート有効時、ブレス・環境音だけの録音チャンクからは再生に何も流れず、スキップされたことがログ／テレメトリで確認できる。
- [ ] ノイズ単独チャンクは GPU 変換を行わずにスキップされる。
- [ ] 発話チャンク内のブレス・環境音区間が出力で抑えられ（既定では完全ミュート）、発話部分の音質・音量は従来と変わらない。
- [ ] 区間の切り替わりでクリック等のアーティファクトが出ない。
- [ ] transcription 側の挙動（フィラー保持を含む）に一切影響がない。
- [ ] ゲート無効（既定）では従来と完全に同一の出力になる。
- [ ] ゲート有効かつモデルが不在・ロード失敗のときは起動時に明示的に失敗する。
- [ ] 変換中の推論で例外が起きても、そのチャンクの声は消えずに出力される。
