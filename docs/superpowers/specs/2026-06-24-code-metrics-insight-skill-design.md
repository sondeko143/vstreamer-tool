# Design: `code-metrics` insight skill

## 問題

コード複雑度を単一指標で見ると「どこをリファクタすれば効くか」が答えられない。分岐は多いが読める dispatcher と、本質的に絡まったコードを取り違えてしまう。既存のゲート系ツールは合否を出すが、リファクタの費用対効果の当たりをつける材料にはならない。

## ゴール

uv ベースの Python プロジェクトのコード複雑度を調査し、「リファクタ候補のランク付き一覧」を各候補の理由付きで出す、再利用可能な on-demand skill。「ゲートは通ったか」ではなく「どこを直すと効くか」に答える。

## 非ゴール

- ゲートではない（合否を出さず、コミット/CI をブロックしない）。
- コードを編集しない（report-only）。
- 総合グレード（A–F の文字評価）を出さない。
- git 履歴トレンドや SaaS 型の解析は範囲外。

## 受入基準

- 任意の uv プロジェクトで、対象パッケージ名をハードコードせずに走る。
- 各関数について複数の複雑度レンズ（少なくとも循環的・認知的）が結合され、1 行に並ぶ。
- リファクタ候補がランク付けされ、各候補に「なぜ候補か」の説明が付く。
- 候補がバケットラベル（both-high / high-CCN-only / high-cognitive-only）で区別される。
- 非ゼロ終了コードを返さず、コミット/CI をブロックせず、コードを編集しない。
- Windows 日本語コンソールでもクラッシュせずに結果を出す。
- 機械可読（JSON）出力も選べる。
