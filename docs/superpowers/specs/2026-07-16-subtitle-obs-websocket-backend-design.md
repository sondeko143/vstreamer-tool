# subtitle の OBS バックエンド: obs-websocket 経由で Text ソースへ push 設計

## 問題

字幕表示が Tk ウィンドウに固定されている。subtitle は vspeech で唯一 tkinter に依存する worker であり、ディスプレイの無い環境（Linux / Docker / サービス実行）では有効にできない。

そのウィンドウを OBS に取り込むには、win32 の `-transparentcolor` によるカラーキーで背景を抜く必要があり、アンチエイリアス画素がキー色と混ざって輪郭に縁が残る。

さらに描画はパイプラインと同じ asyncio ループ上の 30fps ループで走る。アイドル時も 1 コアの約 2.3% を消費し、行が折り返す設定（窓幅を狭める / `font_size` を上げる / `max_text_len` を上げる）では 1 回の再描画がイベントループを 26〜87ms 止める（実測）。これらはいずれも設定で踏める。

## ゴール

- ディスプレイの無い環境で字幕を OBS に表示できる。
- 見た目と挙動が Tk 版と同じであり、設定ファイルが唯一の権威であり続ける（GUI が編集できる項目が実際に効く）。
- 音声パイプラインの生死が OBS の起動状態から独立する。
- 既存の設定ファイルが無改変でこれまで通り動く。

## 非ゴール

- Tk バックエンドの挙動変更。ただし `anchor = "center"` が右寄せになるバグは、移植で発覚したため両バックエンドで同時に直す（"center" が "e" を含み、位置決めと違って justify だけが `== "center"` を先読みしていなかった）。
- OBS のシーン / ソースの自動生成、および配置の操作。
- Windows 以外の OBS（`Text (FreeType 2)`）への対応。
- Tk バックエンドの折り返し性能の改善。

## 受入基準

- [ ] subtitle を有効にしたパイプラインが tkinter を一切必要とせずに起動し、字幕が OBS に表示される。
- [ ] 既存の設定ファイル（バックエンド未指定）は、挙動も表示も変わらない。
- [ ] 設定のフォント・文字色・輪郭色・anchor・margin・ウィンドウ寸法・背景色を変更すると、OBS 上の表示に反映される。
- [ ] 表示時間・履歴数・文字数上限・区切り文字・position によるパネル振り分けが、同じ入力に対して Tk 版と同じ結果になる。
- [ ] `anchor` の 9 値すべてについて、Tk 版と OBS 版が同じ寄せ方を選ぶ（`"center"` は中央寄せになる）。
- [ ] OBS が起動していない状態でもパイプラインが起動でき、後から OBS を起動すると字幕が表示される。
- [ ] 配信中に OBS を再起動しても音声パイプラインは動き続け、字幕表示は自動で復帰する。
- [ ] パスワードが誤っている、または指定したソースが OBS に存在しない場合、警告を出し続けるのではなく、原因を示して終了する。
- [ ] 接続先やソース名が設定されていない場合、起動時に問題を列挙して終了する。
- [ ] 設定を reload すると、OBS 上の表示スタイルが追従する。
- [ ] 背景を透過に設定した場合、輪郭にカラーキー由来の縁が出ない。
- [ ] 字幕の更新が、同一プロセスの他の worker を数十 ms 単位で待たせない。
