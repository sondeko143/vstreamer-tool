# ストリーミング VC の入力エンベロープ追従

## 問題

ストリーミング VC の出力は RVC デコーダ(NSF)の生出力そのままで、入力音声の音量エンベロープに追従しない。有声区間になると出力がほぼフルスケールへステップ的に立ち上がり、無声へ移ると急に落ちるため、口の動きに対して変換音のアタック(立ち上がり)とディケイ(減衰)が急峻に聞こえる。発話系(発話単位バッチ変換)は入力音声の相対ラウドネス包絡を出力へ転写し、実声の自然なアタック/ディケイに追従させているが、ストリーミング経路にはその機構が無い。VAD ゲートのパラメータ(閉時ゲイン・ハングオーバ)調整はゲート由来の急峻さ(閉時ハードミュート等)を和らげるだけで、本質である「入力エンベロープに追従しない」ことは残る。加えて、ストリーミングは 160ms 程度の固定ブロックが単独で到着するため、発話単位のバッチが使う「発話全体の平均で正規化する」手法はそのままでは使えない。

## ゴール

- ストリーミング VC の出力音量が入力音声の相対ラウドネス包絡に追従し、アタック/ディケイが発話単位バッチ変換に近い滑らかさになる(opt-in)。
- 単一ブロックしか手に入らない制約下でも、ブロック内・ブロック跨ぎの双方の音量ダイナミクスに追従する。
- マイク入力ゲインに依存しない(絶対レベルではなく相対形状で追従する)。
- 既定では無効で、有効化しなければ導入前と出力がビット単位で同一である。

## 非ゴール

- 発話系(バッチ)VC の音量整合ロジックの変更。バッチは現状維持。
- ストリーミングコア(固定ブロック+左文脈+クロスフェード)の変換アルゴリズムそのものの変更。
- 出力を持ち上げる方向の整形。既定はダック(減衰)のみで、ヘッドルーム前提の boost は opt-in の代償付き。
- VAD ゲートの置き換え。エンベロープ追従は発話中のアタック/ディケイ整形、ゲートは無音時のノイズ抑制で役割が異なり、両者は併用する。
- 出力ブロックと入力ブロックの微小な時間ずれ(位相合わせ・特徴抽出の受容野に由来)の補償。

## 受入基準

- ストリーミング VC で、出力の音量が入力の相対ラウドネス包絡に追従する(opt-in、既定無効)。
- 立ち上がり(アタック)・減衰(ディケイ)が、機能無効時より滑らかで、発話単位バッチ変換に近い。
- マイク入力ゲインを変えても、追従の形状が保たれる(絶対レベルに依存しない)。
- 無音・語間で出力が入力の静けさに追従して下がり、有声区間はフルスケール付近を保つ。
- 機能を無効にしたとき、本変更の前後で出力はビット単位で同一である。
- ストリーミング系を有効化しても、発話系(文字起こし/翻訳/字幕/発話単位 VC)の挙動は導入前と同一である。
- VAD ゲートと併用でき、両者を有効にしても破綻しない。
