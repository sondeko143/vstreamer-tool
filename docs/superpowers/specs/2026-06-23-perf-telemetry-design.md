# 性能テレメトリ（終了時サマリ＋真E2E）設計書

## 問題

現状のログは人間向けライフサイクルログで性能テレメトリではない。相関 ID が無く、各段の処理時間が不統一で、キュー待ちは未計測、`asctime` ベースのためマシン間クロック skew に弱い。性能改善の PDCA を回すための計測基盤が無い。

## ゴール

各段の処理時間と、発話停止から再生機で音が鳴るまでの真 E2E 遅延を軽量に計測し、プロセス終了時に自動サマリ（段ごと count/p50/p95/max）を出力する。真 E2E は trace_id と原点時刻を wire で伝搬し終点で実測する。外部スタック不要で自己完結させる。

## 非ゴール

- リアルタイムダッシュボード・Prometheus/OpenTelemetry 連携。
- キュー待ち時間・送信時間/サイズの計測。
- 各段合算による近似 E2E。
- NTP 同期自体の自動化（運用前提として文書化し、コードは skew 検知の警告のみ）。

## 受入基準

- 各段の処理時間がプロセス終了時に count/p50/p95/max として集計出力される。
- 真 E2E（発話停止→再生機で音が鳴るまで）が終点で実測され、集計に含まれる。
- trace_id と origin_ts がプロセス／マシンを跨いで wire 伝搬される。
- trace 伝搬がルーティングの等価判定に影響しない（既存のルーティングテストがグリーン維持）。
- クロック skew が疑われるサンプル（e2e<0 または閾値超）は集計から除外され、警告される。
- `enable=false` で計測が完全に no-op になる。
- 保持サンプル数に上限があり、メモリが無制限に増加しない。
