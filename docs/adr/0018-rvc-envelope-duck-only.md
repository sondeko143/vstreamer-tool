# 0018. エンベロープゲインをダック(≤1.0)に限定し max_gain 既定を 1.0 にする

- Status: Accepted
- Date: 2026-07-04
- Related: spec [2026-07-04-rvc-input-envelope-design.md](../superpowers/specs/2026-07-04-rvc-input-envelope-design.md); [ADR-0017](0017-rvc-input-envelope-shape-transfer.md)

## Context

[ADR-0017](0017-rvc-input-envelope-shape-transfer.md) のシェイプ転写を実装したところ、実機で声の大きい部分に音割れ（クリップ）が発生した。RVC 出力は既に int16 フルスケール近く（ピーク ±30000 等）まで出るため、相対抑揚を対称にブーストする（`gain = shape_in` を `max_gain=4.0` まで許す）と平均より大きいフレームが int16 レンジを超えてハードクリップする。数値再現では 1864 サンプルが ±32767 に張り付いた。フルスケール信号は原理的にブースト不可であり、設計書本文が既定 `max_gain=4.0`（対称ブースト）のまま陳腐化していた。

## Decision

エンベロープゲインを **ダック（≤1.0）に限定** し、`max_gain` の既定を **4.0 → 1.0** に変更する。`|out·gain| ≤ |out| ≤ 32767` が常に成り立ち、大きい入力の瞬間は RVC のレベルのまま、静かな瞬間だけ下げる。`max_gain>1` は出力にヘッドルームがある場合のみ（クリップは自己責任）。本 ADR がこの値・方向についての単一情報源であり、旧設計書本文の `max_gain=4.0`（対称ブースト）は無効とする。

## Alternatives rejected

- **`max_gain=4.0`（平均より大きい部分も対称にブースト）** — フルスケール出力をクリップさせる（実測で 1864 サンプルが ±32767 に張り付き）。

## Consequences

大きい入力の瞬間は RVC レベルのまま保たれ、静かな瞬間だけダックされる。これ以前の設計書本文（対称ブースト・既定 4.0）は陳腐化しており、正は本 ADR のダック専用・既定 1.0 とする。
